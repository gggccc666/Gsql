需求分析书

一、总体目标
用C++20/23写一个微型关系型数据库，C/S架构，TCP通信，类似MySQL的命令行交互。不用STL容器，CMake构建，Linux下跑。

二、要实现的功能

【DDL】

- create database <名>
- drop database <名>
- use <库名>
- create table <表名> (列名 类型 [primary], ...)
  类型：int / string（定长256，UTF-8）
  primary列自动建B+树索引
- drop table <表名> （删表+索引文件）

【DML】

- insert <表> values (v1, v2, ...)  字符串用双引号
- select <列/*> from <表> [where 列 =/</> 值]
  有索引就用索引
- delete <表> [where 列 =/</> 值]
- update <表> set 列=值 [where 列 =/</> 值]
  无条件=全表

【界面】

> 提示符，输入SQL，返回结果。exit退出。

三、架构
客户端(CLI) --TCP/JSON--> 服务端
服务端：解析器 -> 执行引擎 -> 存储引擎 -> 文件系统

四、存储结构
data/<库名>/<表名>.dat
data/<库名>/<表名>.idx  （有主键才有）

五、自实现组件

- Vector（替代vector）
- B+树（索引）
- 可能需要：HashMap或类似结构

六、核心类

- Column：列名、类型、是否主键
- Row：存一行的值
- Table：列定义 + 行容器 + 索引指针
- Index：B+树封装
- Database：库名、表集合
- Parser：SQL解析
- Executor：执行逻辑
- StorageEngine：文件读写
- 网络层：Server/Client，JSON序列化

七、约束

- 禁STL容器（string、智能指针、线程等可用）
- C++20以上，Linux，CMake
- 必须C/S分离
- 必须单元测试

八、文件交付

- 报告docx
- 评分表
- 源码目录（.cpp/.hpp/数据文件/CMakeLists.txt）
- config.md（OS、编译环境说明）
- 打zip包，命名为小组名.zip
